# Readability / De-AI Workflow

Goal: produce a readable/publishable candidate while preserving canon.

## Inputs

- Source draft.
- Project state and recent summary.
- Continuity/name/object locks.
- Model routing roles.

## Process

1. Copy or reference source draft; never overwrite it casually.
2. Identify anchors: strong lines, event order, character voice, object states, ending hook.
3. Use side-miners for diagnosis and alternatives.
4. Keep changes grounded; do not invent new plot facts.
5. Final canon role approves final text.
6. Write readable output.
7. Verify canon consistency and count.

## Quality states

- `PASS_UNCHANGED`: original is already best.
- `PASS_REVISED`: revision improves readability without drift.
- `FAIL_NEEDS_REVISION`: not publishable yet.
